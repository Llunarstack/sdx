**Release Date:** March 23, 2026  
**What's New:** Multiple training objectives (flow matching, bridge loss), preference learning (DPO), knowledge distillation, and significantly improved documentation and tooling.

## In One Sentence

v0.2.0 adds alternative training methods beyond VP diffusion, enables preference-based learning from feedback, and provides better code organization and documentation.

---

## What You Can Now Do

### 1. Flow Matching Training
Train models with a simpler, faster approach than traditional diffusion:

```bash
python train.py --data-path my_images/ --flow-matching-training
```

**Why it matters:** Flow matching often converges 20% faster with similar quality.

### 2. Preference Learning (DPO)
Train models to generate images you prefer over alternatives:

```bash
python scripts/tools/training/train_diffusion_dpo \
  --data-path my_preferences/ \
  --reference-ckpt baseline.pt
```

Provide pairs of (good, bad) images with the same prompt, and SDX learns to prefer the good ones.

### 3. Knowledge Distillation
Shrink a large model using a smaller "student" model trained from a larger "teacher":

```bash
python scripts/tools/training/train_kd_distill \
  --student-ckpt small_model.pt \
  --teacher-ckpt large_model.pt
```

**Why it matters:** Smaller models are faster to train and run.

### 4. Bridge Auxiliary Loss
Add a regularization term to improve schedule adherence:

```bash
python train.py --data-path my_images/ --bridge-aux-weight 0.1
```

This helps the model follow the noise schedule more consistently.

## Compatibility

**Good news:** v0.1.0 checkpoints work perfectly with v0.2.0. You don't need to retrain.

**Important:** Flow-trained models must use flow sampling (not VP sampling). Make sure to use matching flags:
- Train with `--flow-matching-training` → Sample with `--flow-matching-sample`
- Train without flow flag → Sample without flow flag

## Getting Started

### 1. Choose Your Training Objective
- **Not sure?** Use `--flow-matching-training` (newer, faster)
- **Want to optimize for your preferences?** Use DPO
- **Want a smaller model?** Use knowledge distillation

### 2. Prepare Your Data
Images + text captions in a folder, or a JSON manifest file.

### 3. Train
```bash
python train.py --data-path my_images/ --flow-matching-training --num-epochs 20
```

### 4. Sample
```bash
python sample.py --ckpt results/best.pt \
  --prompt "your description" \
  --flow-matching-sample \
  --out result.png
```

## Advanced Features

### Speculative CFG
Speed up generation by 20-30% with a draft model:
```bash
python sample.py --ckpt model.pt \
  --speculative-draft-cfg-scale 5.0 \
  --speculative-close-thresh 0.8
```

### Preference Optimization
Guide generation toward your preferred style using CLIP feedback.

### OT Noise Coupling (Experimental)
Couple noise and latents via optimal transport for better denoising paths.

## CLI Reference

```bash
# Training
--flow-matching-training        # Use flow matching objective

--flow-solver heun              # ODE solver (euler or heun)
--speculative-draft-cfg-scale   # Draft CFG for speedup
```

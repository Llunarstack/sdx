## Executive summary

v0.2.0 is a **major capability and documentation release** built on the same core loop (**`train.py` → checkpoint → `sample.py`**). Since v0.1.0 the tree gained **~47k inserted lines** across **320+ files**: new **diffusion objectives** (rectified-flow-style training, VP bridge auxiliary loss, OT-coupled noise), a **matched flow sampler**, **speculative CFG** and other **inference research hooks**, **preference / distillation tooling**, a **refactored `utils/` layout** with focused packages, an expanded **native** stack (CUDA + Python shims), a new **`toolkit/`** training-QoL package, **ViT** and **data** upgrades, and **large doc / blueprint** additions. This document summarizes **what changed**, **why it matters**, and **how to use** the new pieces.

## 1. Diffusion & training objectives (in depth)

### 1.1 VP path (unchanged default)

The default remains **variance-preserving (VP) DDPM** with **`GaussianDiffusion.training_losses`**: ε / v / x₀ prediction, **min-SNR** (and variants), optional **refinement** timestep sampling, **MDM** masked path, etc. v0.2.0 adds **orthogonal** options you can enable explicitly.

### 1.2 Rectified-flow-style **training** (`flow_matching_training`)

- **Module:** [`diffusion/flow_matching.py`](../../diffusion/flow_matching.py)  
- **Objective:** For each batch, sample discrete **`t ∈ {0,…,T−1}`**, map **`s = t/(T−1)`**, build  
  **`x_t = (1−s)·x₀ + s·ε`**, target velocity **`v* = ε − x₀`**, minimize MSE between **`model(x_t, t)`** and **`v*`**.  
- **CLI:** `train.py --flow-matching-training`  
- **Why:** This is a **linear bridge** in latent space (same family as rectified-flow *training* sketches). It is **not** the same object as VP ε-prediction: **do not** expect a VP-trained checkpoint to work with the flow sampler, or vice versa, unless you train and sample consistently.

### 1.3 **Matched flow sampler** (inference)

- **Implementation:** [`GaussianDiffusion._sample_loop_flow_matching`](../../diffusion/gaussian_diffusion.py) and **`sample_loop(..., flow_matching_sample=True)`**.  
- **Dynamics:** Integrate **`dx/ds = vθ(x, t(s))`** from **`s = 1 → 0`** with **`t(s) = round(s·(T−1))`**, matching the **same index map** as training. **Euler** or **Heun** (`flow_solver`).  
- **CLI:** `sample.py --flow-matching-sample`, **`--flow-solver euler|heun`**, **`--force-vp-sample`** to force VP when the embedded config says flow-trained.  
- **Optional API:** `flow_init_noise=` supplies the latent at **`s=1`** (default: i.i.d. Gaussian).  
- **Limitations:** **Structured inpaint** that relies on VP **`q_sample`** trajectories is **incompatible** with the flow flag (validator / runtime guards). **Dual-stage**, **hires-fix**, and the default **`p_step` refine** are skipped or adjusted when flow sampling is active.

### 1.4 **VP bridge auxiliary loss** (`bridge_aux_weight`)

- **Module:** [`diffusion/bridge_training.py`](../../diffusion/bridge_training.py)  
- **Idea:** Form **`x₀' = (1−λ)·x + λ·shuffle(x)`** (batch shuffle pairing), then apply **standard VP `training_losses`** on **`x₀'`** as an **auxiliary** term. Exposes the denoiser to **interpolated endpoints** between different training examples (lightweight “bridge” regularizer—not a full Schrödinger bridge).  
- **CLI:** `--bridge-aux-weight`, `--bridge-aux-lambda`  
- **Composes** with VP main loss, flow main loss, or MDM main loss (validator still forbids **flow + MDM** together).

### 1.5 **OT noise–latent coupling** (`ot_noise_pair`)

- **Module:** [`utils/training/ot_noise_pairing.py`](../../utils/training/ot_noise_pairing.py)  
- **Idea:** Within a batch, optionally **couple** Gaussian noise to latents via **entropic OT** (Sinkhorn) or a **Hungarian** assignment (when SciPy is available). Intended as an experimental **denoising-path** / **matching** prior—not a full optimal-transport generative model.  
- **CLI:** `--ot-noise-pair-reg`, `--ot-noise-pair-iters`, `--ot-noise-pair-mode soft|hungarian`

### 1.6 **Per-sample training losses & spectral SFP**

- **`GaussianDiffusion.per_sample_training_losses`** refactors how **weighted / curriculum** losses are built; **spectral SFP** (FFT-weighted error) remains a **VP-side** prototype—see [`diffusion/spectral_sfp.py`](../../diffusion/spectral_sfp.py) and **MODERN_DIFFUSION**.

### 1.7 **Linear latent bridge helper**

- [`diffusion/latent_bridge.py`](../../diffusion/latent_bridge.py) — simple **linear interpolation** between latents for experiments / tooling (distinct from **`bridge_training`** VP aux loss).

### 1.8 **Config validation**

- [`utils/training/config_validator.py`](../../utils/training/config_validator.py) enforces **flow vs MDM** exclusivity, **bridge** weight / λ bounds, and existing OT / MDM / REPA rules.

## 2. Training loop (`train.py`) — behavioral summary

- **Flow branch:** When `--flow-matching-training`, the main loss uses **`flow_matching_per_sample_losses`**; **OT-paired ε** is reused when OT is enabled, else fresh Gaussian noise.  
- **Bridge add-on:** If `--bridge-aux-weight > 0`, a **second VP loss** runs on shuffled-mix latents (independent noise for that term).  
- **Validation:** **`eval_val_loss`** mirrors the same **flow / MDM / VP + bridge** logic and passes **OT-coupled noise** where applicable for consistent metrics.  
- **New CLI flags** (non-exhaustive): `--flow-matching-training`, `--bridge-aux-weight`, `--bridge-aux-lambda`, `--ot-noise-pair-*` (see **Train CLI** in [`README.md`](../../README.md)).

---

## 3. Sampling (`sample.py`) — behavioral summary

- **Flow:** As above; img2img / from-z use **linear flow bridge** **`(1−s)x₀ + sε`** when flow sampling is on (instead of VP **`q_sample`**), except where **mask inpaint** forces VP semantics.  
- **Speculative CFG:** [`utils/generation/speculative_denoise.py`](../../utils/generation/speculative_denoise.py) — optional **draft CFG scale** + **accept / blend** when full and draft predictions are close. Flags: `--speculative-draft-cfg-scale`, `--speculative-close-thresh`, `--speculative-blend`.  
- **CLIP monitor:** Periodic decode / **CLIP cosine** vs prompt to **boost CFG** when similarity is low (expensive; opt-in).  
- **Volatile CFG:** Spike detection on latent updates → temporary CFG boost.  
- **Research hooks:** e.g. **spectral latent blend**, **layout / domain** priors — see [`utils/generation/inference_research_hooks.py`](../../utils/generation/inference_research_hooks.py) and [`utils/generation/clip_alignment.py`](../../utils/generation/clip_alignment.py).  
- **SAG, reference tokens, pick-best, face enhance** (from v0.1.0 trajectory) remain; README lists **combined** sampling flags.

## 4. Preference learning & distillation

### 4.1 Diffusion DPO (stage-2 style)

- **Script:** [`scripts/tools/training/train_diffusion_dpo.py`](../../scripts/tools/training/train_diffusion_dpo.py)  
- **Pattern:** Load a **policy** checkpoint, freeze a **reference** copy, optimize a **DPO-style** objective from **paired win/lose images** and a **shared prompt**, using **shared `t` and noise** across the pair for coupled VP losses.  
- **Triple checkpoints:** Uses **`load_text_encoder_bundle`** so **`text_encoder_mode=triple`** gets **T5 + CLIP-L + bigG + fusion** at encode time (not T5-only shortcut).

### 4.2 Teacher–student KD (same architecture)

- **Script:** [`scripts/tools/training/train_kd_distill.py`](../../scripts/tools/training/train_kd_distill.py)  
- **Loss:** **MSE** between **student** and **frozen teacher** outputs on the **same** **`x_t, t`** (VP **`q_sample`**), optional **`--from-scratch`** student init.

### 4.3 Supporting modules

- [`utils/training/diffusion_dpo_loss.py`](../../utils/training/diffusion_dpo_loss.py)  
- [`utils/training/preference_image_dataset.py`](../../utils/training/preference_image_dataset.py)  
- [`utils/training/preference_jsonl.py`](../../utils/training/preference_jsonl.py)

## 5. Repository layout refactor (`utils/` and friends)

v0.2.0 continues the **package-oriented** layout: functionality moved from flat **`utils/*.py`** into **namespaced subpackages** (e.g. **`utils/generation/`**, **`utils/architecture/`**, **`utils/checkpoint/`**, **`utils/consistency/`**, **`utils/modeling/`**, **`utils/training/`**) so imports and ownership are clearer. **Public entry points** remain **`train.py`**, **`sample.py`**, and **`inference.py`**; internal modules import from the new paths.

**Impact for downstream forks:** Any **hard-coded** `from utils.foo import bar` may need updating to the new module path—grep the tree or run **`pytest`** / **`python scripts/tools/dev/smoke_imports.py`** after merging.

## 15. Upgrade notes (from v0.1.0)

1. **Pull** latest `main` (or checkout **`v0.2.0`**).  
2. **Reinstall** deps: `pip install -r requirements.txt` (+ **`requirements-cuda128.txt`** on NVIDIA).  
3. **`python -m toolkit.training.env_health`** — verify CUDA / torch.  
4. **If you imported old `utils.*` paths**, update imports or use **`python scripts/tools/dev/smoke_imports.py`**.  
5. **Checkpoints:**  
   - **Flow-trained** models → use **flow sampling** (or explicit CLI).  
   - **VP-trained** models → keep **default VP** sampling.  
6. **Native:** Rebuild **`native/cpp`** if you rely on ctypes symbols after C++ changes.

# SDX v8 (source release)

Version **8** is the **creativity + prompt parity** release: invent original aesthetics instead of recycling artist tags, one unified **PromptStack v2** for training and inference, and a cleaned **native** layer (Rust / Go / CUDA / Mojo) for style exploration at scale.

**Versioning:** Docs and README brand this line as **v8**. `pyproject.toml` uses **8.0.0** (PEP 440). Git tag: **`v8.0.0`** (or **`v8`**).

---

## Game-changing features (v8)

### 1. Style Genome ŌĆö invent styles, donŌĆÖt copy names

Structured **orthogonal aesthetics** (palette, line, surface, camera, lighting, signature) compile to positive / negative / style-channel strings. Novelty checks avoid near-duplicate genomes in a bank.

```bash
python sample.py --ckpt results/.../best.pt \
  --prompt "lone samurai at dusk" \
  --invent-styles 1 --style-inventor-mode insane --style-chaos-level 0.85 \
  --out out.png
```

**Explore at scale:** invent genomes ŌåÆ JSONL manifest ŌåÆ batch sample or dedupe for training packs.

```bash
python -m scripts.tools explore_styles --prompt "void priest" --mode apocalypse --chaos 1.0
python -m scripts.tools explore_styles --list-presets
```

Modules: `utils/prompt/style_genome.py`, `style_inventor.py`, `style_explore.py`, `style_genome_chaos.py`, `style_memory.py`.

### 2. PromptStack v2 ŌĆö one pipeline, train = sample

All post-user prompt assembly runs through **`utils.prompt.stack`**: intelligence ŌåÆ style genome ŌåÆ guidance ŌåÆ negatives ŌåÆ content controls ŌåÆ clauses ŌåÆ neg filter. Training captions reuse the **guidance** stage via `merge_guidance_for_training_caption()`; `data/caption_utils.py` delegates to the stack.

```bash
python -m scripts.tools preview_prompt_stack --prompt "1girl, portrait" --quality-pack top
SDX_PROMPT_STACK_TRACE=1 python -m scripts.tools preview_prompt_stack --prompt "..." --json
```

See [`docs/PROMPT_STACK.md`](../PROMPT_STACK.md).

### 3. Chaos / fusion / hypermutation

Eight insane presets, **genome fusion**, **hypermutation**, and invention modes (`normal`, `insane`, `apocalypse`, `chimera`, `glitch`, `eldritch`, `cyberpunk`). Prompt clauses: `style.chaos`, `style.surreal`, `style.glitch`, `style.apocalypse`.

`sample.py`: `--explore-styles`, `--explore-styles-insane`, `--style-genome-fusion`, `--style-genome-hypermutate`, `--style-genome-preset`.

### 4. Native style ops (optional, Python fallbacks always)

| Layer | What |
|-------|------|
| **Rust** `sdx-prompt-ops` | Jaccard, FNV fingerprint, multi-axis merge (`style_ops_native`) |
| **CUDA** `style_pick_best.cu` | Best embedding index vs query (`cuda_style_pick_native`) |
| **Go** `sdx-manifest` | `explore-stats`, `explore-dedupe` on style JSONL |
| **Mojo** `sdx_style_tokens.mojo` | Token merge + fingerprint experiments |

**Layout fix:** all `.cu` sources live under **`native/cpp/cuda/`** only; `native/cuda/` removed. Clean script: `scripts/tools/native/clean_native_builds.ps1`.

```bash
python -m scripts.tools test_style_native_stack
python -m scripts.tools explore_styles --show-native
```

## v7 ŌåÆ v8 comparison

| Area | v7 | v8 |
|------|----|----|
| **Prompt assembly** | Mostly ad-hoc chain in `sample.py` / caption utils | **PromptStack v2** stages + traceable preview CLI |
| **Training vs inference text** | Partial overlap | **Shared guidance stage**; caption shortcomings/neg parity fixes |
| **Style creativity** | Artist/tag banks (`STYLE_ARTIST_TAGS`) | **Style Genome** invention + explore manifests + chaos modes |
| **Sampling flags** | Holy Grail, TCIS, pick-best | + `--invent-styles`, explore shortcuts, genome fusion/hypermutate |
| **Native** | CI + eval + general fast paths | + **style-specific** Rust/CUDA/Go/Mojo; **native tree cleanup** |
| **Operator focus** | Reproducibility, CI, baseline eval pack | **Agentic originality** + same reproducibility baseline |

**Still from v7 (unchanged baseline):** Ruff + basedpyright CI, Dependabot, `run_artifacts`, baseline eval pack, research playbooks, `SECURITY.md`.

## Upgrade from v7

1. Pull / checkout tag **`v8.0.0`**.
2. `pip install -r requirements.txt` (unchanged entry).
3. Optional native rebuild:
   ```powershell
   .\scripts\tools\native\build_native.ps1
   cd native\rust\sdx-prompt-ops; cargo build --release
   ```
4. Try style exploration without a checkpoint:
   ```bash
   python -m scripts.tools explore_styles --prompt "test" --dry-stack
   ```

# SDX v5

Release highlights for **inference test-time scaling**, **data curation**, **stabler Diffusion DPO**, and **ViT tooling**.

## Inference (sample.py)

- **Pick-best metrics:** `vit`, `combo_vit`, `combo_vit_hq`, `combo_vit_realism`, `combo_count_vit`, `aesthetic`, `aesthetic_realism`, plus existing `combo_*` names.
- **`--pick-best auto`:** when `--pick-vit-ckpt` is set, auto selects ViT-aware metrics; `--pick-auto-no-clip` prefers cheap / non-CLIP paths where possible.
- **Beam search:** `--beam-width`, `--beam-steps`; micro-beam `--beam2-*`. Default beam scoring uses `combo_vit_hq` if a ViT checkpoint path is provided.
- **CLIP monitor rewind:** `--clip-monitor-rewind` with periodic CLIP scoring.
- **`--pick-report-json`:** machine-readable scores for beam / final pick (debug + preference mining).

## Diffusion core

- **`GaussianDiffusion.sample_loop`:** `return_intermediate_state` for beam resume semantics.
- **Soft rewind** on low periodic alignment when configured.

## Data

- **`manifest_enrich`:** `aesthetic_proxy`, optional `clip_sim`, optional `dedupe_phash`.
- **`data_quality`:** `--min-clip-sim`, `--min-aesthetic-proxy`.

## Training tools

- **`train_diffusion_dpo`:** `--dpo-logit-clip`, `--sync-ref-every`.
- **`vit_quality`:** `--timm-kwargs` JSON; checkpoint restores `timm_kwargs`.

## Version

- Package version in `pyproject.toml`: **5.0.0** (marks this source refresh; training compatibility unchanged unless you opt into new flags).

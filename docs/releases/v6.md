# SDX v6.0.0

Release highlights since **[v5.0.0](v5.md)**: **native fast paths**, **sampling extras layout**, **book / visual memory**, **developer tooling** (Pyright / clangd / compile flags), and **research sketches**.

## Layout and import paths

- **`vit_quality/`** is the canonical ViT quality training stack (the old top-level **`ViT/`** package is removed). Use `python -m vit_quality.train`.
- **`utils/architecture/`** and **`utils/generation/`** are the canonical utility trees (historical `utils/arch` / `utils/gen` shims are retired or documented under `DEPRECATIONS.md` where still relevant).
- **Holy Grail companion modules** now live under **`diffusion/sampling_extras/`**. The **`diffusion.holy_grail`** package remains a **thin compatibility re-export**; prefer `from diffusion.sampling_extras import ...` in new code.

## Native acceleration

- **`native/python/sdx_native`**: NumPy- and buffer-oriented helpers (scans, caption CSV, timestep grids, CHW pack, diffusion sigma/SNR fast path, attention masks, histograms, etc.). Many symbols are also reachable via **`from utils.native import ...`**.
- **Optional C** under **`native/c/`**: `sdx_c_buffer_stats` (newline + byte sum) and uint8 HWC **image metrics**; Python loads DLLs/SOs from the same **`native/cpp/build/...`** convention as other native artifacts when built.
- **Editor / static analysis**: root **`pyrightconfig.json`**, **`.clangd`**, **`compile_flags.txt`**, and **`native/cpp/compile_flags.txt`** so `sdx_native`, **`#include "sdx/..."`**, and **`native/c`** headers resolve consistently.

## Diffusion and sampling

- **`diffusion/sampling_extras/`**: blueprint, condition annealing, guidance fusion, latent refiner, presets, recommender, runtime guard, style router (the former in-repo “spex” / holy_grail submodule layout consolidated here).
- **`diffusion/snr_utils.py`**: SNR helpers with optional **`sdx_native`** acceleration after Rust paths.
- **`diffusion/pixel_perfect`**: pixel-alignment oriented helpers for sampling / post (see module docstring).

## Book / comic pipeline

- **Visual memory** (`visual_memory.py`, `visual_memory_bridge.py`) plus book helpers: manifest validation, prompt intel, style fusion and authenticity, challenging content, text continuity, model readiness.
- **Tools**: `book_manifest_check`, `book_prompt_audit`, `visual_memory_patch`; example **`examples/book_visual_memory.example.json`**.

## Config and prompts

- **`config/defaults/style_artists.py`** / **`style_guidance.py`**: richer style tag facets and automatic merging of tag-quality hints into style guidance (used from `sample.py` / caption utilities).
- **New defaults**: **`creature_character_prompts.py`**, **`physics_material_prompts.py`** for creature / physics-aware prompts.

## Generation utilities

- **`utils/generation/simple_latent_generate.py`**, **`latent_edit_helpers.py`**, **`dit_ar_latent_compat.py`**: small, test-covered helpers for latent I/O, edits, and DiT/AR compatibility.

## Research

- **`research/`**: non-production sketches (autoregressive plans, noise structures, hybrid schedules, latent agreement, timestep weights, etc.) with focused unit tests.

## Tooling and CI

- **`.github/workflows/ci.yml`**: **Ruff** + **pytest** on push and pull request.
- **`scripts/tools`**: entrypoints grouped by domain; obsolete one-off scripts removed where superseded.

## Tests

- Added tests for **book** modules, **sampling_extras**, **native** fast paths, **pixel_perfect**, **visual memory bridge**, **checkpoint validation**, **research** sketches, and stricter **test-time pick** tolerances when native weighted-sum is enabled.

## Version

- **`pyproject.toml`**: **6.0.0** (marks this source drop; training compatibility is unchanged unless you opt into new modules or paths).

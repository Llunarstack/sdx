# SDX v7 (source release)

Version **7** is a **framework and operator-quality** release: stronger CI, reproducibility hooks, evaluation scaffolding, security reporting, and research maps—not a single pretrained product checkpoint.

**Package / tag:** `pyproject.toml` and Git tag **`v7.0.0`** both denote this **v7** source drop (the README refers to this line as **v7**, not patch-level marketing).

---

## Highlights

### Reliability and CI

- **GitHub Actions:** Ruff, **basedpyright** (errors-only) on curated modules, **`verify_doc_links`**, pytest.
- **Dependabot** (`.github/dependabot.yml`): weekly updates for Actions and pip.
- **Optional pre-commit** (`.pre-commit-config.yaml`): Ruff check + format locally.
- **`.editorconfig`**: tracked so editor defaults and `docs/FILES.md` links stay consistent.

### Reproducibility and eval

- **`utils/generation/run_artifacts.py`**: canonical `run_manifest.json` / `config.train.json` names; **`train.py`** uses them.
- **`utils/generation/inference_stages.py`**: ordered stage ids aligned with the README system diagram (for logging/UI/docs).
- **`examples/eval_prompts_baseline.json`** + **`utils/generation/eval_prompt_pack.py`**: JSON prompt packs for regression-style runs.
- **`examples/run_baseline_eval.py`**: dry-run or `--execute` batch over `sample.py`.

### Docs and recipes

- **`docs/recipes/quick_eval_holy_grail.md`**: demo + `sample.py` + Holy Grail + training manifests.
- **`docs/recipes/local_ci_mirror.md`**: mirror CI locally.
- **`docs/recipes/eval_baseline_prompts.md`**: baseline eval pack + driver.
- **`docs/research/IMAGE_QUALITY_LEVERS_2026.md`**: 2025-26 literature mapped to SDX hooks (CFG, flow, data, DPO).
- **`docs/research/SAMPLING_EXPERIMENTS_BACKLOG.md`**: CFG / steps / solver experiment grids.

### Security

- **`SECURITY.md`**: vulnerability reporting and operator scope notes.

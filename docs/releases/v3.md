# SDX v3 — Automated Quality Improvement and Benchmarking

**Release Date:** April 4, 2026  
**What's New:** Automated hard-case detection, benchmark-driven training loops, and production-readiness checklists.

## In One Sentence

v3 automates finding your model's weaknesses (hard cases) and improves them systematically, plus helps you start a project with confidence.

---

## What You Can Now Do

### 1. Detect Hard Cases Automatically
Identify which types of prompts your model struggles with:

```bash
python -m scripts.tools benchmark_suite \
  --ckpt results/best.pt \
  --suite-pack top_contender \
  --seed-list 42,123,999 \
  --export-hardcases-jsonl hardcases.jsonl
```

**What happens:** 
- Runs your model on 100+ test prompts
- Scores images with multiple metrics
- Identifies failures: text in images, counting errors, blur, exposure problems
- Saves hard cases to a JSON file

### 2. Improve Automatically
Set up a loop that finds problems, fixes them, and measures improvement:

```bash
python -m scripts.tools auto_improve_loop \
  --base-ckpt results/base.pt \
  --iterations 3 \
  --export-hardcases \
  --promote-best
```

**What happens:**
1. Benchmarks your model
2. Identifies hard cases
3. Extracts preference pairs from failures
4. Trains DPO on those failures
5. Re-benchmarks and compares
6. Repeats 3 times
7. Saves the best result

### 3. Startup Readiness Check
Before spending hours training, verify everything is set up:

```bash
python -m scripts.tools startup_readiness \
  --out-json report.json \
  --out-md report.md
```

**What it checks:**
- Python packages installed correctly
- GPU available and working
- Disk space available
- CUDA properly configured
- Gives next steps if issues found

## Why This Matters

**Before v3:** You'd train for hours, then manually test and find problems. Fixing them meant starting over.

**After v3:** You can automatically identify weaknesses and iteratively improve your model with each training loop.

## Key Features

### Robust Benchmarking
Instead of "your model got 75% on one test," you now get:
- Multiple seed runs (different random numbers)
- Stability score (does it always work or is it lucky?)
- Saturation-aware quality scoring
- Failure categorization

### Hard Case Driven Training
Focus training on what actually fails:
- Text in images not working? Train on that
- Character counts wrong? Train on that
- Motion blur? Train on that

### Auto-Improve Loop
Fully automated improvement:
```
Benchmark → Find failures → Mine preferences → Train DPO → Benchmark again → Repeat
```

## Understanding Robustness Scores

```
Score = Mean Quality - (Penalty × Std Dev)
```

This means:
- Higher is better
- Penalizes inconsistency
- Finds models that reliably work, not just lucky ones

**Example:**
- Model A: Mean=80, Std=5 → Score = 80 - (0.15 × 5) = 79.25
- Model B: Mean=75, Std=1 → Score = 75 - (0.15 × 1) = 74.85
- Model A wins despite lower average (more reliable)

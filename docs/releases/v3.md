## Executive summary

v3 focuses on production-minded quality iteration loops and startup readiness:

- More reliable checkpoint ranking through **seed-robust benchmarking**
- Automated **hard-case mining** (text/count/blur/exposure/oversaturation failure tags)
- **Hardcase-aware preference mining** to over-focus DPO training on weak spots
- Multi-iteration **auto-improve loop** with optional best-checkpoint promotion
- A no-training **startup readiness report** to audit environment and launch blockers

---

## Highlights

### 1) Benchmark robustness and quality scoring

- `scripts/tools/benchmark_suite.py`
  - Supports `--seed-list` and `--robustness-penalty`
  - Ranks by robust score (`mean - penalty * std`) instead of single-run luck
  - Adds saturation-aware quality scoring
  - Exports hardcases via `--export-hardcases-jsonl`

### 2) Hard-case driven improvement loop

- `scripts/tools/training/mine_preference_pairs.py`
  - Hardcase-aware pair mining:
    - `--hardcases-jsonl`
    - `--hardcase-extra-pairs`
    - `--hardcase-min-margin-scale`
- `scripts/tools/ops/auto_improve_loop.py`
  - Iterative loop support (`--iterations`)
  - Hardcase export and hardcase-aware preference re-mining
  - Optional best-checkpoint promotion

### 3) Startup preflight (no training required)

- `scripts/tools/ops/startup_readiness.py`
  - Audits packages, GPU/CUDA, native acceleration visibility, key files, disk headroom, and optional dataset manifest
  - Emits machine-readable JSON and optional markdown summary
  - Gives blockers/warnings plus suggested next commands

## Key commands

```bash
# Readiness report (before you start any expensive training)
python -m scripts.tools startup_readiness \
  --out-json startup_readiness_report.json \
  --out-md startup_readiness_report.md

# Benchmark one or more checkpoints with robust ranking + hardcase export
python -m scripts.tools benchmark_suite \
  --ckpt results/a/best.pt results/b/best.pt \
  --suite-pack top_contender_proxy_v1 \
  --seed-list 42,123,999 \
  --robustness-penalty 0.15 \
  --export-hardcases-jsonl bench/hardcases.jsonl

# Full auto-improve loop (benchmark -> mine prefs -> DPO -> re-benchmark)
python -m scripts.tools auto_improve_loop \
  --base-ckpt results/base/best.pt \
  --iterations 3 \
  --seed-list 42,123,999 \
  --export-hardcases \
  --promote-best \
  --promote-path auto_improve_loop/best_auto.pt
```

## Notes

- v3 does not claim universal SOTA by itself; it adds a repeatable system to measure and improve toward that goal.
- All core paths still work without native accelerators; native components remain optional.
